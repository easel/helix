Consolidate helix status into a single surface answering: (a) current cycle number, (b) currently-executing bead id and claim age, (c) parked beads classified by reason (dep-blocked, retry-parked with next-eligible-time, orphaned-claim), (d) recently-injected supervisory beads (review-finding, alignment-review) with triggering cycle and triggering bead, (e) stop reason of the last run. No new ddx subcommand.&#xA;&#xA;Parallel-run mutex: second &#39;helix run&#39; on the same project root must detect an active run (via DDX_PROJECT_ROOT-scoped advisory lock or existing .helix-run.pid) and return exit code 2 with stderr &#39;helix run: already running (pid=&lt;N&gt;, started=&lt;ts&gt;)&#39; BEFORE claiming any bead.&#xA;&#xA;Migration observability counters (written to .ddx/run-state.json or a sibling file on each helix run): ddx_delegation_success_count, ddx_delegation_failure_count with reason classes (missing_binary, too_old, malformed_json, timeout), parallel_run_denied_count, supervisory_injection_count by type.
    </description>
