## Intent Summary

- **Vision**: HELIX should act as a supervisory autopilot layered on DDx, with the tracker as the steering wheel and DDx owning managed execution while HELIX owns workflow semantics and interpretation (`docs/helix/00-discover/product-vision.md:117-136`).
- **Requirements**: Repo users should default to sparse-intent intake plus DDx-managed queue draining, while retained HELIX wrappers stay thin, transitional, and artifact-governed (`docs/helix/01-frame/prd.md:33-46`, `docs/helix/01-frame/features/FEAT-011-slider-autonomy.md:82-89`, `docs/helix/01-frame/features/FEAT-011-slider-autonomy.md:204-223`).
- **Features / Stories**: FEAT-011 defines the reduced public execution surface and DDx handoff, while FEAT-007 and the active site/demo concerns require public docs and demo reels to stay synchronized with current workflow rules (`docs/helix/01-frame/features/FEAT-011-slider-autonomy.md:141-180`, `workflows/concerns/hugo-hextra/practices.md:82-94`).
- **Architecture / ADRs**: CONTRACT-001 and TD-011 assign queue-drain and close-with-evidence mechanics to `ddx agent execute-loop` / `ddx agent execute-bead`, leaving HELIX responsible for stage stance and workflow routing rather than duplicate model-version policy or wrapper-owned bookkeeping (`docs/helix/02-design/contracts/CONTRACT-001-ddx-helix-boundary.md:178-188`, `docs/helix/02-design/contracts/CONTRACT-001-ddx-helix-boundary.md:213-216`, `docs/helix/02-design/technical-designs/TD-011-slider-autonomy-implementation.md:214-230`, `docs/helix/02-design/technical-designs/TD-011-slider-autonomy-implementation.md:279-305`).
- **Technical Design**: `helix align` remains a bead-governed prompt launcher; when a governing bead is created or materially updated, digest refresh and deterministic area scoping are part of the contract, not optional embellishments (`workflows/actions/reconcile-alignment.md:123-150`, `workflows/references/context-digest.md:65-68`).
- **Test Plans**: Deterministic shell harnesses and validators should prove the wrapper contract, context-digest propagation, and skill wording invariants without relying on undocumented shims or brittle exact-string guards (`docs/helix/01-frame/prd.md:125-130`, `workflows/actions/fresh-eyes-review.md:46-48`, `workflows/actions/report.md:35-38`).
- **Implementation Plans**: The repo implementation plan is a queue snapshot and is currently current, but the build-phase workflow bundle still contains legacy tracker-path wording that would generate stale guidance in downstream repos (`workflows/phases/04-build/README.md:52-60`, `workflows/phases/04-build/artifacts/implementation-plan/template.md:27-35`).
