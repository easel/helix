# Launch Checklist Generation Prompt

Create a cross-functional launch readiness checklist ensuring all teams are prepared for a coordinated product launch.

## Storage Location

Store the launch checklist at: `docs/helix/05-deploy/launch-checklist.md`

## Purpose

The launch checklist serves as:
- A cross-functional coordination tool
- A single source of launch readiness truth
- A go/no-go decision framework
- A risk identification mechanism
- A launch day reference

## Key Requirements

### 1. Product Readiness

#### Feature Completeness
- All launch features deployed
- Known issues documented
- Feature flags configured
- Performance validated
- Accessibility verified

#### Quality Assurance
- Test coverage complete
- UAT sign-off received
- Bug triage complete
- Edge cases handled
- Localization verified

### 2. Marketing Readiness

#### Content Assets
- Website updates live
- Blog post scheduled
- Social content queued
- Email campaigns ready
- Press materials prepared

#### Campaigns
- Paid campaigns configured
- UTM tracking verified
- Landing pages tested
- Forms connected to CRM
- Analytics dashboards ready

### 3. Sales Enablement

#### Training
- Sales team briefed
- Demo scripts prepared
- Objection handling ready
- Competitive positioning shared
- FAQ documented

#### Materials
- Sales deck updated
- One-pagers created
- Case studies ready
- Pricing confirmed
- Contract templates updated

### 4. Support Readiness

#### Documentation
- Help center updated
- Knowledge base articles
- Video tutorials
- API documentation
- Migration guides

#### Operations
- Support team trained
- Escalation paths defined
- SLA commitments confirmed
- Monitoring alerts configured
- On-call schedule set

### 5. Legal/Compliance

#### Documentation
- Terms of service updated
- Privacy policy reviewed
- Data processing agreements
- Regulatory compliance verified
- Third-party agreements signed

#### Risk Management
- Legal review complete
- IP considerations addressed
- Export compliance checked
- Accessibility compliance
- Security certifications valid

### 6. Go/No-Go Decision

#### Decision Framework
- Launch criteria defined
- Stakeholder alignment
- Risk tolerance established
- Rollback triggers identified
- Communication plan ready

#### Escalation Path
- Decision makers identified
- Escalation timeline
- Delay communication plan
- Partial launch options

## Template Guidelines

### Ownership
- Each item has clear owner
- Backup owners assigned
- Escalation contacts listed

### Status Tracking
- Binary completion status
- Blockers highlighted
- Dependencies mapped
- Timeline visibility

## Quality Checklist

Before finalizing the launch checklist:
- [ ] All functions represented
- [ ] Critical path identified
- [ ] Owners assigned
- [ ] Timeline realistic
- [ ] Dependencies clear
- [ ] Blockers escalated
- [ ] Go/no-go criteria defined

## Integration with Deploy Phase

This checklist coordinates with:
1. **GTM Plan** - Marketing and sales activities
2. **Deployment Checklist** - Technical readiness
3. **Release Notes** - Customer communication

Remember: A launch is a team sport. Every function must be ready for the launch to succeed.
