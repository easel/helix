# Story Iteration Report Generation

Create a post-deployment iteration report for a single user story. Capture outcomes, learnings, and follow-up work concisely.

## Focus
- Compare expected outcomes against production reality.
- Capture only the evidence, lessons, and follow-up work that change future execution.
- Turn follow-up work into tracker issues, not prose wish lists.

## Completion Criteria
- All acceptance criteria are reviewed with actual outcomes.
- Follow-up work is captured as backlog issues.
- Required canonical updates are identified.
